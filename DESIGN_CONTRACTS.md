# KBO 매니저 — 연봉/계약 구조 설계 (3단계-1)

> 3단계 구단 운영의 첫 블록. **진짜 목적은 제도 재현이 아니라 '선수 가치 평가
> 함수'** — 능력치·나이·potential을 받아 적정 연봉/계약가치를 산출한다.
> 이 함수가 있어야 드래프트(지명 가치)·FA(입찰가)·트레이드(가치 비교)가 다 돌아간다.
>
> ※ 이 문서는 **제안**이다. 승인 후 구현 (에이징 때와 동일 절차).

## 0. 원칙 / 범위

- **시즌 간 로직**: 오프시즌에만 작동 (에이징 `offseason_tick` 직후 체인).
  단일 시즌 경기 지표에 영향 없어야 한다 (회귀 가드, §7).
- **핵심만**: 래리버드룰·FA등급제(A/B/C 보상)·옵션계약은 이번 범위 제외 (나중에).
- 상수는 관례대로 `TUNE["contract"]` 한 곳 (에이징이 `TUNE["aging"]`을 둔 것과 동일).
- 돈 단위는 **억원**(기존 `Contract.salary`와 동일).
- `random.Random(seed)` 주입 → 재현 가능.
- **로드맵 위치**: 연봉/계약(지금) → 드래프트 → FA → 트레이드. 드래프트가 2번이라
  이번 설계는 **지명권을 평가 가능한 자산으로 취급할 훅**을 남긴다 (§6, 구현은 드래프트 단계).

## 1. 선수 가치 평가 함수 — 이 단계의 심장

야구 연봉 평가 표준인 **WAR × ($/WAR)** 접근. 능력치를 종합 생산지표(WAR)로
환산한 뒤 금액으로 매긴다. 나이·potential은 **미래가치 투영**으로 자연 반영된다.

### 1.1 능력치 → 기대 WAR (단일 시즌)

```
war_now = k · (OVR − OVR_repl) · role_pt · scarcity(pos)
```

- `OVR`: 기존 `bat_overall`/`pit_overall` 재사용.
- `OVR_repl`: **연봉** 대체선수 수준(≈55). 이하 = 0 WAR → 최저연봉. 에이징의 은퇴
  대체(42)와는 **별개 개념**(자유롭게 구할 수 있는 선수 수준). 로스터 평균≈58
  대비 55로 잡아야 리그 총연봉이 캡 근처가 된다 (§7 캘리브레이션 확정값).
- `k`: WAR 스케일(설계값, 예: 야수 0.15 → OVR 80 주전이 ≈5.7 WAR). 투수는 별도 `k_p`.
- `role_pt`: 출전 기대치. 주전 1.0 / 준주전 0.6 / 벤치 0.3, 선발 1.0 / 불펜 0.4.
  (로스터 내 OVR 순위로 역할 추정 — 실제 라인업 로직과 동일 정렬 재사용.)
- `scarcity(pos)`: 포지션 희소성 배율. C·SS·CF 프리미엄(>1), 1B·DH·코너 할인(<1).

### 1.2 미래가치 투영 (나이·potential 반영의 핵심)

에이징 커브의 **노이즈 제외 기대 궤적**(`aging.expected_delta`, 재능 g·d 반영)을
재사용해 앞으로 H년간 기대 WAR을 추정하고 할인 합산한다:

```
asset_war = Σ_{t=0..H}  war(age+t) / (1 + r)^t        # r=할인율(~0.10), H=투영 지평(~6년)
```

- 젊은 선수는 잔여 전성기가 길어 `asset_war`가 크다 → **"31세 OVR 80(잔여 짧음)"과
  "24세 OVR 76(pot 82, 잔여 김)"의 가치가 자동으로 갈린다** (요구사항 직해결).
- `war_now`는 "올해 적정 연봉", `asset_war`는 "다년계약/트레이드/드래프트 가치"에 쓴다.

### 1.3 금액 환산 — $/WAR

```
dollar_per_war = 리그캡 × cap_to_war_ratio     # 캡에 연동 → 캡 인플레가 연봉에 자동 전파
적정연봉(억)   = max(최저연봉, war_now × dollar_per_war)
계약가치(억)   = asset_war × dollar_per_war
```

- `$/WAR`를 리그 캡에 연동(설계값 `cap_to_war_ratio`)해, §3 캡이 매년 오르면 연봉
  수준도 함께 오른다 (별도 인플레 상수 불필요, 드리프트 위험 축소).
- 하한 = 최저연봉(신인/대체선수급도 바닥은 받음).

## 2. 계약 산정

- **다년 연수**: 나이·`asset_war` 기반 휴리스틱. 젊고 pot 크면 장기(최대 ~5년),
  노장이면 단기(1~2년). (래리버드/옵션 없이 단순화.)
- **AAV 산정 (캡 반영)**: `AAV = (연봉 + 계약금) / 계약연수`. 캡에는 AAV로 잡힌다
  (현행 KBO 다년계약 규정). 계약금은 총액의 일정 비율(설계값).
- `Contract`에 `signing_bonus`(계약금)·`total_value` 파생 추가 (§ 데이터 모델).

## 3. 경쟁균형세(소프트캡) + 하한

- **소프트캡**: 상위 40명 AAV 총액 상한. **하드캡 아님** — 초과 가능하나 제재.
  - 캡 값: 2025 **137억**, 2026~28 **매년 +5%**. 시뮬은 `cap_year0` + `cap_growth`(0.05)로
    매 오프시즌 상향.
  - 초과 시: 제재금(초과분 × `luxury_rate`) 예산 차감 + **드래프트 지명권 페널티**
    (지명권 자산에 `penalty` 플래그 — §6 훅, 실제 적용은 드래프트 단계).
- **하한(최소 지출)**: 상위 40명 총액이 `floor`(캡의 일정 비율) 미달 시 강제 지출
  또는 페널티. 바닥 구단이 지출을 안 해 영구 리빌딩하는 것을 막음 (런어웨이 하방 차단).

## 4. 동적 예산 (FM식, 런어웨이 방지가 핵심 함정)

구단 예산이 성적·수입으로 매시즌 변동하되, **리그 캡(천장)+하한(바닥) 안에 갇힌다**.

```
target_budget = base_market(tid) + revenue_gain(전시즌 성적·관중)      # 성적→수입→예산
budget = clamp( ema(budget_prev, target, α),  floor,  cap )           # 완만 이동평균 + 상하한
```

- **런어웨이 방지 3중 장치** (설계의 핵심):
  1. **캡/하한 하드 클램프**: 아무리 벌어도 캡 위로, 아무리 못해도 하한 밑으로 못 감.
  2. **이동평균(EMA)**: `α`(~0.3)로 완만 이동 — 한 시즌 성적으로 예산 폭변 금지.
  3. **변동폭 클램프**: 시즌당 ±`max_swing`%(설계값) 이상 못 움직임.
- `base_market`: 구단별 시장 크기(수도권 큰손 vs 지방 짠물) **고정 오프셋**. 단:
  - **(a) 완만한 격차**: 최상단↔최하단 기본예산 차이를 **20~30% 수준**만 (KBO는
    캡 137억이 있어 MLB식 대격차 리그가 아님).
  - **(b) 동적 변동이 지배적**: 시장차는 '출발점의 완만한 차이'일 뿐, **성적 기반
    동적 변동이 시장차를 넘어설 만큼 커야** 한다. 잘하는 짠물 구단이 몇 시즌
    성공하면 큰손 예산 수준까지 올라올 수 있어야 함 — **시장차가 유리천장이 되면
    안 됨**(그러면 동적 예산의 의미가 반감). §7-5 고착 검증으로 확인.
- 강팀이 벌어→더 강해지는 양의 피드백을, 캡 수렴 + 하위팀 하한 보장으로 상쇄.
  (다만 진짜 전력 재분배 축인 **드래프트 역순지명은 다음 단계** — §7 검증 한계 명시.)

## 5. 구단 지출 성향 AI (뼈대만)

각 구단이 예산 안에서 얼마나 적극적으로 쓸지 결정하는 스칼라 + 약점 진단.

```
aggressiveness ∈ [0,1] = f( 캡 여유,  약점 포지션 수,  팀 단계 )
team_phase = 윈나우(상위 순위 + 노장 코어) | 리빌딩(하위 + 젊은 코어) | 중립
weak_positions = [pos: 해당 포지션 최고 OVR이 리그 중앙값 미달]
```

- **지금은 여기까지** — `aggressiveness`와 `weak_positions`를 산출하는 함수만.
  실제 FA 입찰·트레이드 행동은 다음 단계들이 이 출력을 소비한다.

## 6. 드래프트 지명권 자산화 훅 (구현 X, 인터페이스만)

드래프트가 2번으로 앞당겨졌으므로, 지명권을 **선수와 동급의 '평가 가능한 자산'**으로
다룰 여지를 지금 남긴다.

- `Team.draft_picks: list[DraftPick]` 필드 **예약** (round, year, original_tid, penalty).
- 가치평가를 **자산 디스패치**로 설계: `value_of(asset)` 가 Player면 §1, DraftPick이면
  "기대 신인 `asset_war`"(역순지명 곡선 × `generate_prospect` 기대값)로 분기하도록
  **인터페이스만** 열어둔다. DraftPick 평가의 실제 구현은 드래프트 단계.
- 이렇게 하면 FA 보상·트레이드가 "선수 ↔ 지명권"을 같은 통화(WAR/억원)로 비교 가능.

## 7. 검증 계획 — `scripts/economy_check.py`

1. **연봉 분포 리얼리즘**: 리그 전체 연봉 히스토그램이 실제 KBO스러운지 —
   소수 고액 스타(수십억) + 다수 저연봉 + 신인 최저연봉. 롱테일 형태 확인.
2. **가치평가 sanity 표**: 실존 선수를 넣어 평가액이 직관적인지 —
   김도영(젊은 특급, 높은 asset_war) · 최형우(노장 스타, 높은 war_now·낮은 asset_war) ·
   스텁 신인(최저 근처)을 표로 나란히.
3. **런어웨이 방지 (최우선)**: 30시즌 연속 시뮬에서 특정 구단이 영구 지배하거나
   영구 바닥에 깔리지 않고 순위가 시간에 걸쳐 순환하는지 (순위 자기상관/우승 분산).
   **※ 한계 명시**: 드래프트 역순지명이 아직 없어 이 검증은 **'돈 축'만** 본 것 —
   드래프트 단계에서 '돈+역순지명' 두 축으로 **재검증 예정**임을 문서·출력에 표기.
4. **회귀 가드**: 계약/예산은 시즌 간 로직 → 단일 시즌 경기 지표 불변
   (`test_contracts.py`로 가드, 에이징의 "시즌 중 무변화" 테스트와 동형).
5. **시장차 고착 확인**: 시장차를 뒀으므로 특히 중요 — 30시즌에서 큰손 구단이
   만년 상위·짠물 구단이 만년 하위로 **고착되지 않는지** (구단별 순위/예산 시계열).
   고착되면 시장차 폭 축소 또는 동적 변동폭 확대.
6. **장기 연봉 인플레 확인**: `$/WAR`를 캡(+5%/yr)에 연동했는데 리그 WAR 총량은
   대체로 일정 → `$/WAR`가 매년 올라 연봉이 실질 인플레될 수 있음. **1시즌 vs
   30시즌 연봉 분포를 나란히** 놓고 인플레가 현실적 수준(실제 KBO 연봉 상승률대)인지
   과도한지 확인. 과도하면 `$/WAR` 연동을 캡 상승률보다 완만하게 조정.

### 확정된 핵심 결정 (검토 완료)
1. **WAR × $/WAR** 가치 모델 + 에이징 궤적 재사용한 **미래가치 할인 합산** (나이/pot 반영)
2. **$/WAR를 캡에 연동** (연봉 인플레를 캡 성장에 자동 종속 → 드리프트 축소)
3. **동적 예산 = EMA + 캡/하한 + 변동폭 클램프** 3중 런어웨이 차단
4. **구단별 시장 크기 기본차 있음** — 완만한 20~30% 오프셋, 동적 변동이 지배적
   (유리천장 아님), 캡·하한에 눌려 격차 유한
5. **지명권 자산화는 `value_of` 디스패치 훅만**, 구현은 드래프트 단계
6. 지출 성향은 **스칼라+약점진단 뼈대만**
