# KBO 매니저 — 신인 드래프트 설계 (3단계-2)

> 로드맵 2번. 연봉/계약(3단계-1) 위에 얹는다. **진짜 목적은 전력 재분배의
> 두 번째 축**(돈에 이은 역순지명)과, 스카우팅 불확실성으로 만드는 FM식
> 드래프트 긴장감이다.
>
> ※ 이 문서는 **제안**이다. 승인 후 구현 (지금까지와 동일 절차).

## 0. 원칙 / 범위 · 이미 있는 것 활용

- **시즌 간 로직**: 오프시즌 체인 `에이징(은퇴) → 드래프트 → 재정`. 단일 시즌
  경기 지표 무영향 (회귀 가드).
- **재사용 (신규 생성 아님)**:
  - `generate_prospect(rng, is_pitcher, pos)` → 드래프트 풀 생성에 그대로.
  - `Player.tal_g/tal_d` 숨김 재능 → **스카우팅 불확실성의 관측 대상**(§3).
  - `asset_war` / `value_of` → 지명 가치 통화. `DraftPick`·역순 훅 → 순서·자산화.
- 상수는 `TUNE["draft"]` 한 곳.
- **핵심 구조 변경**: 현재 `aging.offseason_tick`은 은퇴자마다 스텁 신인을 1:1
  자동 생성한다. 드래프트가 이 자동 대체를 **대체**한다 — 은퇴는 로스터에
  '구멍'만 남기고, 그 구멍을 드래프트가 풀에서 채운다 (generate_prospect 정식화).

## 1. 지명 순서 — 순수 역순 (Z자, KBO식)

- 전 시즌 순위 **역순**: 꼴찌 1순위 … 우승팀 마지막. 추첨 없음.
- **모든 라운드 동일 역순** (Z자, 스네이크 아님). 총 **11라운드, 팀당 최대 11명**.
- **패스 허용**: 로스터가 25명이 되면 그 팀은 이후 라운드 지명 스킵. 구멍 많은
  팀이 더 많이 뽑아 자연스럽게 refill.

## 1b. 신인 풀 생성 (실제 KBO 반영)

실제 KBO는 지명대상 1,100~1,260명 중 ~110명(10%)만 지명되고 90%는 탈락하며,
후보 모집단 자체가 **투수 편중**(아마야구 투수 보직 다수)이다. 게임 반영:

- **(a) 풀 규모**: 1,200명을 다 만들지 않는다(90%는 저능력 → 시뮬 낭비). '지명
  가치 있는 상위 풀'만 = 리그 은퇴 구멍(~110) + 경계선 여유분 = **150~200명**.
  `size = clamp(총구멍 × pool_factor(1.5), 150, 200)`. 미지명자는 **폐기**
  (실제 90% 탈락과 일치; 2군/재수 캐리오버는 이후 육성 시스템).
- **(b) 포지션 편중 (모집단 단계)**: 후보 풀을 **투수 60~65% / 야수 35~40%**로
  두껍게. 뽑힌 결과가 아니라 모집단이 이미 투수 편중. 효과: 투수는 풀이 두꺼워
  골라 뽑을 여유, 야수 키포지션(C·SS)은 희소해 좋은 자원이 나오면 상위 지명
  → Need 로직이 현실적으로 작동.
- **(c) 은퇴 보정**: (b) 기본 편중 위에 리그 은퇴 구멍 분포를 `retire_blend`만큼
  섞는다. 특정 포지션 대량 은퇴 시 그 포지션 공급 소폭↑(기본 투수편중은 유지).
- 생성기는 **`generate_prospect`(아키타입 3티어) 그대로** — 포지션만 위 분포로
  샘플링해 넘긴다.

## 2. 지명 결정 — Need 우선 + 가치 보조 (핵심)

한 지명에서 팀 `T`가 라운드 `r`에서 잔여 풀의 유망주 `p`에 매기는 점수:

```
score(T, p, r) = scouted_val(p)  +  need(T, pos(p)) × need_max_bonus × need_mult(r)
                                  +  ceiling(p, r)
```

**라운드별 Need 차등** (실제 KBO 단장 패턴: 두산/NC/kt/LG 사례 — 상위 BPA·중위
니즈픽·하위 실링). 단일 가중치가 아니라 라운드에 따라 Need 무게가 바뀐다:
- **상위(1R)** `need_mult 0.3`: Need 약화 → 가치 지배(BPA). 프랜차이즈 특급이
  Need 무시하고 뽑힘 (검증: 약점포지션 지명 11~20%, 1R 특급 SP 다수).
- **중위(2~3R)** `need_mult 1.4`: Need 피크 → 약점 포지션 집중 보강(니즈픽).
  (검증: 약점포지션 지명 50~52%.)
- **하위(4R~)** `need_mult 0.4` + `ceiling`: Need 약화 + 실링 베팅. `ceiling`은
  젊을수록↑(관측 가능한 업사이드 프록시 — 진짜 재능 g는 숨김이라 직접 못 봄,
  §3 스카우팅 불확실성 유지). (검증: 평균나이 19.8, 상위/중위 20.6~20.7보다 젊음.)

> ※ 우리 드래프트는 은퇴 구멍만 refill(팀당 ~2픽)이라 KBO 11라운드 클래스보다
> 얕다(픽 대부분 1~3R). 그래서 KBO의 1-2/3-7/8+ 라운드 매핑을 **실제 깊이에
> 맞춰 1/2-3/4+로 축소**했다. `round_mid_start`·`round_late_start`로 조정.
> 전체 약점포지션 지명률은 ~30%로 유지(변화 전 31%) — 얕은 드래프트에서 픽이
> 1R(BPA)에 몰려 전체는 BPA에 눌리고, Need 강화는 **중위 구간에 집중** 발현.

- `scouted_val(p)`: **스카우팅 관측 자산가치** (진짜 asset_war에 노이즈, §3).
- `need(T, pos) ∈ [0, 1]`: 그 포지션 뎁스 부족도. 구멍/약점=1, 강점=0.
  `need = clamp((need_ref − depth(T,pos)) / need_span, 0, 1)`
  - `depth(T,pos)` = 그 포지션에서 잔류할 최고 OVR (구멍이면 0).
  - `need_ref`(~62, '준수한 주전')·`need_span`(~15)는 튜닝.
  - 투수는 SP/RP 뎁스로 분리 (선발 로테이션 5·불펜 충원 필요도).
- `need_max_bonus`: 완전한 구멍이 더하는 **WAR 환산 최대 가중**(~6.0). = Need 무게.

**BPA 예외는 이 덧셈 구조에 내장된다** (별도 분기 불필요, `need_max_bonus` 한 손잡이로 균형):
- *Need 픽*: 특급 아닌 비슷한 값들 사이 → 구멍 포지션이 이긴다.
  예) 포수 유망주 val 12 + need 1.0×6 = **18** vs 외야 val 15 + need 0.2×6 = 16.2 → 포수.
- *BPA 오버라이드*: 프랜차이즈 스타감은 값이 압도적이라 Need를 누른다.
  예) 유격수 특급 val 24 + need 0×6 = **24** vs 포수(구멍) val 12 + 6 = 18 → 특급.
- 균형 튜닝: `need_max_bonus`가 크면 Need 편향(특급 놓침 위험), 작으면 BPA 편향
  (약팀이 남는 포지션만 쌓음). 지명 결과 sanity(§검증 2)로 중간값 탐색.

## 3. 스카우팅 불확실성 — 숨김 재능의 노이즈 관측 (핵심)

구단은 유망주의 **진짜** 미래가치(재능 g·d가 좌우)를 정확히 모른다. 보는 것은
노이즈 낀 관측치다. 이게 "1순위 bust / 5라운드 대박"의 원천.

```
scouted_val(p) = true_asset_war(p) × exp( N(0, scout_noise) )     # 로그정규 승산노이즈
```

- 유망주마다 **1회 추첨된 컨센서스 관측치** (드래프트 보드). 지명은 이 보드 순서를
  Need로 비튼 것 → 진짜 값과 어긋나 고순위 실패·저순위 성공이 확률적으로 발생.
- `true_asset_war`는 숨김 `tal_g/d`를 반영한 실제 잔여가치(§연봉단계 asset_war).
  관측 노이즈는 개념적으로 **재능(g/d)에 대한 불확실성**을 대리한다.
- `scout_noise`(σ, 시작값 ~0.35): 이 한 상수가 재미의 축.
  - 너무 크면 지명 순위가 무의미(완전 랜덤 → 실력 무의미),
  - 너무 작으면 확정적(역전 없음 → 재미 없음).
  - **중간값 탐색**: §검증 3(지명순위 vs 훗날 성공의 상관이 있되 <1)으로 조정.
- (확장 여지: 팀별 스카우팅 편차는 이후. 지금은 컨센서스 보드 + 팀별 Need 만으로도
  팀마다 다른 지명이 나온다.)

## 4. 미결 통합 — 컨택 특화형 (README 알려진 이슈 1)

드래프트가 `generate_prospect`를 정식 풀 생성기로 승격하는 김에, 장기 모드
타율왕 .390+ 상시화(컨택+선구 특화형 18% 티어가 OVR 68로 상시 출전+성장)를
같이 잡는다.

- **조정 후보**: (a) 컨택 계열 시그니처 하향 — 아키타입별 `sig_mean`을 도입해
  contact/eye 시그니처만 86→~78, 또는 (b) 특화형 성장재능 억제(컨택 한정).
- **제약**: '특화형 타이틀 재생산' 효과는 유지 — 대주자형(도루왕)·거포형(홈런왕)은
  그대로 두고 컨택만 눌러 균형점을 찾는다.
- **측정은 반드시 4시드+**: TUNE 변경이 RNG 스트림을 통째로 이동시켜 1%p 효과는
  2시드로 구분 불가 (README 방법론). §검증 5에서 4시드 이상 장기 시뮬로 확인.

## 5. 하한 강제집행 연결 (연봉단계 훅 소비)

연봉단계에서 남긴 하한 미달 플래그(예산 < 캡×floor_frac, 31/300 팀-시즌)를
드래프트가 소비하는 집행 지점:

- 드래프트로 들어온 신인은 계약과 함께 로스터에 편입 → 자연히 지출이 는다.
- 지명·계약 후에도 여전히 하한 미달인 팀은 **floor 톱업**: 남는 예산으로 잔류
  선수 재계약가를 상향(또는 최소지출 배분)해 하한을 채운다. (강제집행의 한 축;
  전면적 지출 AI는 FA 단계.)

## 7. 검증 계획 — `scripts/draft_check.py`

1. **역순 효과**: 꼴찌 팀 지명 유망주 평균 `asset_war` > 우승팀 지명 유망주
   (약팀에 좋은 신인이 몰리는지).
2. **Need 작동 + BPA 예외**: 구단이 약점 포지션을 우선 지명하는 비율 확인 +
   압도적 특급은 Need 무시하고 뽑히는 사례 존재.
3. **스카우팅 불확실성**: 지명 순위와 N년 후 실제 OVR의 **상관 > 0이되 < 1**
   (1순위 성공률 > 5라운드지만 역전 사례 존재). `scout_noise` 조정 근거.
4. **★ 30시즌 순위 순환 (완전판)**: 연봉단계에서 '돈 축만'이라 미룬 검증을
   이제 '돈 + 역순지명' 두 축으로 완성. 특정 구단 영구지배/영구바닥 없이 순위가
   순환하는지 — **이번 최대 검증**. (역순지명이 약팀을 끌어올리는 음의 피드백이
   시장차·성적 양의 피드백을 상쇄하는지.)
5. **컨택 특화형 해결**: **4시드+** 장기 시뮬에서 타율왕 .390+ 상시화 소멸,
   동시에 도루왕/홈런왕 꼬리는 유지.
6. **회귀 가드**: 드래프트는 시즌 간 로직 → 단일 시즌 경기 지표 불변.

### 승인받을 핵심 결정
1. 지명 순서 = **순수 역순, 매 라운드 동일** (KBO식, 추첨 없음)
2. 지명 점수 = `scouted_val + need × need_max_bonus` — **덧셈 구조로 BPA 내장**,
   `need_max_bonus` 한 손잡이로 Need↔BPA 균형
3. 스카우팅 = **컨센서스 보드에 로그정규 노이즈**(`scout_noise` σ~0.35), 재능 g/d
   불확실성 대리 → bust/대박
4. 은퇴 자동 스텁 대체를 **드래프트 refill로 교체** (generate_prospect 정식화)
5. 컨택 특화형은 드래프트 구현 때 **4시드+로** 함께 조정 (도루·홈런 꼬리 유지)
