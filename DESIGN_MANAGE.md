# KBO 매니저 — 감독 모드 설계 (웹 UI MVP-3: 라인업 + 경기 중 개입)

> 범위: 라인업 편집(타순·수비 포지션·선발 로테이션·불펜 역할) + 경기 중 개입
> (투수 교체·대타·대주자·대수비, 관전 중 일시정지). 전제가 되는 **구조 전환
> (라이브 시뮬 + 난수 격리)** 이 이 문서의 심장이다.
>
> ※ 이 문서는 **제안**이다. 승인 후 Part 1(라인업)부터 구현.

## 0. 문제 정의

현재 구조 "하루 일괄 시뮬 → 확정 → 리플레이 관전"에서 경기 중 개입은 원리적으로
불가능하다 (6회에 투수를 바꿔도 결과가 이미 확정). 또한 현재 하루의 모든 경기가
**시즌 RNG 하나를 순차 소비**하므로, 유저 경기의 난수 소비량이 달라지면 같은 날
다른 경기·이후 시즌 전체가 오염된다. 개입을 열려면 격리가 선행돼야 한다.

## 1. ★구조 전환 — 난수 격리 + 라이브 시뮬

### 1.1 난수 격리 (2단계 파생 스트림)

```
시즌 RNG ──(매일 1회, 고정 순서)──▶ day_seed
day_seed ──▶ Random(day_seed)      ──▶ 경기별 game_seed (경기당 1개, 고정 순서)
         ──▶ Random(hash(day_seed, tid)) ──▶ 팀별 일일 틱 스트림 (부상·폼)
```

- **경기별 독립 RNG**: 경기 안에서 무슨 일이 일어나든(개입 포함) 난수 소비가
  그 경기 밖으로 새지 않는다 → 같은 날 다른 경기 불변.
- **팀별 일일 틱 스트림**: 부상/폼 틱도 팀별 독립. 유저 개입으로 유저 팀
  투구수·부상 여부가 달라져도(정당한 결과) **다른 팀** 부상·폼 롤은 불변
  → 이후 시즌의 타 팀 진행 완전 격리.
  (주의: 공유 스트림을 유지하면 유저 팀의 부상 발생 여부가 롤 소비 개수를
  바꿔 전 리그가 오염된다 — 팀별 분리가 격리의 필수 조건.)

### 1.2 모드 정책 — 기존 검증 자산 보존 (핵심 결정)

격리는 난수 소비 패턴을 바꾸므로 **기존 시드 앵커(seed 2026 = KIA 97승)와
호환되지 않는다**. 두 모드로 분리한다:

| 모드 | 사용처 | 재현성 |
|---|---|---|
| **공유 모드** (기존, 기본값) | 콘솔 스크립트·전 검증 하네스·기존 테스트 | seed 2026 = KIA 97승 등 **앵커 전부 보존** |
| **격리 모드** (`SeasonRunner(isolated=True)`) | 웹 세션 전용 | 자체 재현성 (같은 시드 = 같은 시즌). 공유 모드와는 다른 난수 우주 — 새 앵커를 테스트로 핀 |

- 통계 분포는 두 모드가 동일(같은 엔진·같은 확률) → 캘리브레이션·순환 검증
  결과는 모드와 무관하게 유효.
- **가드 재정의**: "개입 안 하면 AI와 동일"이 아니라 **"개입이 다른 경기·
  시즌 시스템을 오염시키지 않음"** + "AI 자동 진행은 (모드별) 시드 재현".

### 1.3 라이브 유저 경기

- `step_day`(격리 모드): 타 팀 4경기는 지금처럼 일괄 시뮬. **유저 경기는
  LiveGame으로 보류** — 자기 game_seed로 시작하되 결정 지점에서 멈춘다.
  유저 경기 종료 후에야 그날이 닫힌다(부상 틱은 유저 경기 투구수 필요).
- **GameSimulator 리팩터**: `run()`의 타석 루프를 `step()`(타석 1개 진행)으로
  분해 — **순수 기계적 분해, 로직·rng 소비 순서 불변** (기존 66개 테스트 +
  시드 앵커가 리팩터 회귀를 가드).
- 관전 화면 통합: 라이브 경기도 **같은 이벤트 스트림 포맷**으로 흘려보낸다
  (Watch 프론트 재사용). 리플레이의 청크 은닉은 라이브에선 자연 해소 —
  결과가 아직 존재하지 않는다. (관전 없이 넘어가는 경로는 기존 스포일러
  통제 유지: AI 위임 일괄 시뮬 → 미공개.)

## 2. Part 1 — 라인업 편집 (경기 전)

- **타순 9명 + 수비 슬롯**(8포지션+DH) 배정. 검증: 9명 중복 없음, 슬롯 완비.
- **포지션 부적합**: 엔진 확인 결과 수비력은 선수 능력치(fielding/arm)로만
  반영되고 '슬롯 적합 페널티'는 없다(포수 arm은 C 슬롯 배치자 기준 — 비포수를
  C에 놓으면 그 선수 송구로 도루 저지가 자연 하락). 이번엔 **경고 표시만**
  (주포지션≠슬롯이면 ⚠) + 능력치 자연 반영. 정식 적합 페널티는 백로그.
- **선발 로테이션**: 5인 순서. **불펜 역할**: 마무리 1 + 셋업 지정(필승조
  우선 투입 힌트 — 엔진 교체 AI가 셋업 목록을 우선 고려), 나머지 추격조.
- **AI 추천 버튼**: `build_default_lineup`/`build_default_pitching` 결과로 리셋.
- 부상자 자동 대체는 현행(`refresh_lineup`) 유지. `user_managed` 훅 소비.
- API: `GET/PUT /api/my/lineup` (타순·슬롯·로테이션·마무리·셋업 + 경고 목록 반환).

## 3. Part 2 — 경기 중 개입

- **정지 지점** (서버 판정, 위임 OFF일 때 자동 정지):
  (a) 하프이닝 사이 (b) 수비 위기 — 주자 2+ 또는 이번 이닝 2실점+ 그리고 투수
  피로(`fatigue_penalty>0`) (c) 대타 기회 — 득점권 주자 + 타순 7~9번
  (d) 유저가 아무 때나 [일시정지] (다음 타석 전에 멈춤).
- **개입 메뉴** (유저 팀 상황에 따라):
  - 수비 중: **투수 교체** — 현재 투수 상태(투구수·피로·실점)와 불펜 목록
    (남은 체력·연투 여부) 제시. 유저 교체 시 그 하프이닝 AI 교체 로직은
    유저 결정을 존중(중복 교체 안 함).
  - 공격 중: **대타**(벤치 목록, 교체된 선수 재출전 불가), **대주자**.
  - **대수비**: 이닝 사이 수비 슬롯 교체.
- **위임 토글**: 기본 ON(AI가 지금처럼 전부 운영, 정지 없음 = 현행 관전과
  동일). OFF면 정지 지점에서 멈춤. 언제든 전환.
- 엔진에 교체 API 추가: `sub_batter/sub_runner/sub_fielder/force_pitcher` —
  로스터 상태 변경일 뿐 확률 로직 불변. 재출전 불가는 used 집합으로.

## 4. 검증 계획

1. **★난수 격리 (4시드+)**: 격리 모드에서 유저 경기에 임의 개입(테스트가
   투수 강제 교체 주입) → 같은 날 **다른 4경기 결과 동일** + **이후 시즌의
   타 팀 성적·부상·폼 동일** (개입 유/무 두 런 비교, 시드 4개).
2. **모드별 재현성**: 공유 모드 seed 2026 = KIA 97승 유지 (기존 테스트).
   격리 모드 자체 앵커 신설(같은 시드 = 같은 순위표).
3. **리팩터 회귀**: `run()`=`step()` 루프 분해 후 기존 66개 테스트 전부 통과
   (단일 경기 시드 앵커 포함 — rng 순서 불변 증명).
4. **개입 효과**: 같은 game_seed로 A/B — (a) 지친 선발 방치 vs 6회 교체
   N회 반복 → 교체 쪽 실점 감소 확인 (피로 시스템 반영 증명),
   (b) 최강 타자 1번 vs 9번 배치 N경기 → 1번 쪽 득점 우위 (타순 반영 증명).
5. **라인업 검증**: 9명/슬롯 완비 강제, 부적합 경고, 부상자 대체 유지.

## 5. 배치 / 단계

| 파일 | 내용 |
|---|---|
| `kbo/league/season.py` | `isolated` 모드 (day_seed 파생, 팀별 틱 스트림) |
| `kbo/engine/game.py` | `step()` 분해(기계적) + 교체 API + 정지 지점 판정 |
| `kbo/models/team.py` | `setup`(셋업 목록) — 교체 AI가 우선 고려 |
| `web/backend` | LiveGame 세션·개입/라인업 API |
| `web/frontend` | 라인업 편집 화면 + 관전 개입 패널 |

- **Part 1**: 격리 모드 + 라인업 편집 + 검증 1·2·3·5 → 커밋.
- **Part 2**: step() 분해 + LiveGame + 개입 UI + 검증 1(개입 주입)·4 → 커밋.

### 승인받을 핵심 결정
1. **2모드 분리** — 공유 모드(콘솔·검증, 앵커 보존) / 격리 모드(웹 전용,
   경기별 seed + 팀별 틱 스트림). 기존 검증 자산을 깨지 않고 격리 확보.
2. 가드 재정의: "개입이 타 경기·타 팀 시즌을 오염시키지 않음" (4시드 검증)
3. `run()`→`step()` 순수 기계적 분해 (rng 순서 불변, 기존 테스트로 가드)
4. 포지션 부적합은 이번엔 경고 + 능력치 자연 반영 (적합 페널티 백로그)
5. 위임 기본 ON — 개입은 옵트인, 정지 지점 3종 + 수동 일시정지
