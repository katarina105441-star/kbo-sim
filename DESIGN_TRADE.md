# KBO 매니저 — 트레이드 설계 (3단계-4, 구단 운영 마지막 블록)

> **본질은 윈나우 ↔ 리빌딩의 니즈 교환** (LG-키움 최원태 트레이드: 윈나우가
> 즉전 선발을 받고, 리빌딩이 유망주 2명+1R 지명권을 받은 윈윈). FA(일방 영입)와
> 근본적으로 다르게 **두 팀이 동시에 이득이어야** 성사된다.
>
> ※ 이 문서는 **제안**이다. 승인 후 구현.

## 0. 원칙 / 범위 · 이미 있는 것 활용

- **오프시즌만**. 체인: `에이징(은퇴) → 트레이드 → FA → 드래프트 → 재정`.
  트레이드가 FA 앞: 약점을 트레이드로 먼저 시도하고, 리빌딩팀이 확보한 지명권이
  뒤 드래프트에서 바로 행사된다. 단일 시즌 무영향(회귀 가드).
- **거래 자산 = 선수 + 지명권만** (현금 트레이드 생략). 등가 판단은 `value_of`
  통일 통화(억원).
- **재사용**: `value_of`/`asset_war`(등가) · `spending_ai`류 윈나우/리빌딩 판별 ·
  `need_frac`(약점) · 재능계수 숨김(유망주 도박성) · `DraftPick`(자산화 훅 소비).
- 상수는 `TUNE["trade"]`.

## 1. 팀 상태 진단 — 방향과 재료

매 오프시즌 각 팀을 진단한다 (기존 로직 재사용):

```
phase(T)  : 윈나우 | 리빌딩 | 중립
  contend = 최근 순위 점수(상위=1) , core_age = 로스터 주축 평균 나이
  윈나우  = contend ≥ 상위권 & core_age 높음 (지금 이겨야 함)
  리빌딩  = contend 하위권 & (core_age 높음 = 팔 베테랑 있음)
wants(T)  : 원하는 것
  윈나우  → 약점 포지션(need_frac 높음)의 즉시전력 (war_now 높은 베테랑)
  리빌딩  → 유망주(젊고 pot 높음) + 지명권
surplus(T): 내놓을 수 있는 것
  윈나우  → 유망주(주전 아님 + 어림) + 지명권
  리빌딩  → 주축 베테랑(war_now 높고 나이 많음, 고연봉이면 더 적극)
  공통    → 뎁스 잉여 포지션(need_frac ≈ 0에 준수한 선수 복수)의 후순위 선수
```

## 2. 양방향 매칭 — 성사 3조건 ★심장

팀 쌍 (A, B)의 패키지 `A가 주는 자산들 ↔ B가 주는 자산들`이 **3조건 전부**
충족해야 성사:

- **(a) 등가성**: 양측 가치 합이 근사 — `|V(A측) − V(B측)| ≤ tol × max(V)`
  (tol ~15%). 크게 기울면 **결렬** = "1라운더 2명은 과하다" 후려치기 거절 재현.
  단 가치는 §3의 **GM 주관 평가**로 잰다 (아래).
- **(b) 니즈 부합 (양쪽 다)**: A가 받는 것이 A의 `wants`에 맞고 **동시에**
  B가 받는 것도 B의 `wants`에 맞아야. 등가라도 니즈가 안 맞으면 결렬
  (즉전이 필요한 윈나우에게 유망주를 줘봤자 거래 안 됨).
- **(c) 방향 상보성**: 매칭 후보를 니즈가 **반대**인 쌍으로 좁힌다.
  윈나우↔리빌딩 = 최고 상보성. 윈나우↔윈나우는 둘 다 유망주를 안 내놔
  성사가 **자연히** 어렵다 (금지 규칙이 아니라 surplus/wants 불일치의 결과).

### 매칭 알고리즘 (오프시즌 1회)

```
1) 전 팀 phase/wants/surplus 진단.
2) 팀 쌍 상보성 점수 = 방향 반대(윈나우×리빌딩) × (A wants ∩ B surplus 크기)
                       × (B wants ∩ A surplus 크기)
   → 점수 높은 쌍부터 시도 (전 쌍 완전탐색 대신 상보 쌍만).
3) 쌍마다 패키지 탐색 (작게):
   - 뼈대: 리빌딩의 베테랑 V ↔ 윈나우의 유망주 P.
     (V는 윈나우의 최대 약점 포지션에서, P는 윈나우의 잉여 유망주 중 최고 pot.)
   - 가치 격차가 tol 초과면 **지명권/보조 유망주를 얹어** 균형 시도
     (1R→2R→3R 순으로 맞는 라운드 선택, 그래도 안 맞으면 결렬).
4) 3조건 검사 → 성사 시 자산 이동, 두 팀 모두 이번 오프시즌 추가 거래 제한
   (팀당 max_trades, 리그 총량 한도) → 다음 쌍.
```

## 3. GM의 가치 판단 — 주관성 (거래가 성사되는 이유이자 도박인 이유)

```
V_gm(T, asset) = value_of(asset) × exp( N(0, gm_noise) )    # 팀×자산별 관측
```

- 드래프트 스카우팅 노이즈와 **같은 메커니즘 재사용** (σ는 더 작게 ~0.20 —
  프로 선수는 아마 유망주보다 정보가 많다).
- 각 팀은 **자기 관점의 가치로** (a) 등가성·이득을 판단: A는 이 유망주를 높게
  보고 B는 낮게 보면 **서로 자기가 이득이라 믿고** 거래가 성사된다 (실제
  트레이드의 성사 원리). 양팀 모두 "내 관점에서 받는 것 ≥ 주는 것 × (1−ε)"
  이어야 함.
- **도박성**: 받은 유망주의 진짜 미래는 숨김 재능계수(g/d)가 결정 — 몇 년 후
  대박(이주형 케이스)일지 bust일지 갈린다. 검증 §4에서 추적.

## 4. 지명권 트레이드 — 훅의 실구현 (구조 변화 1개)

지금까지 지명권은 자산화 훅만 있었다. 트레이드가 이를 실물로 만든다:

- **민팅**: 트레이드 단계 시작 시 **다음 드래프트의 1~3라운드 지명권**을 팀별로
  생성 (`DraftPick(year+1, round, original_tid)`). 4R 이하는 가치가 낮아 거래
  대상에서 제외(민팅 안 함).
- **가치 (`value_of` 디스패치 구현)**: `pick_value[round] × $/WAR` — 라운드별
  기대 신인 asset_war 테이블(1R ~9, 2R ~6, 3R ~4 WAR, 검증으로 조정) ×
  미실현 리스크 할인. 경쟁균형세 `penalty` 플래그면 추가 할인.
- **드래프트 연동**: `run_draft`가 지명 슬롯 순서(원소유팀의 역순 위치)는
  유지하되 **그 슬롯의 지명을 지명권 보유팀이 행사**. 로스터 25 상한과 자연
  정합: 리빌딩팀은 베테랑을 내줘 구멍+지명권을 얻고 → 그 슬롯에서 유망주를
  더 뽑는다. 윈나우팀은 유망주를 내주고 베테랑으로 채워 로스터 중립.

## 6. 검증 계획 — `scripts/trade_check.py`

1. **건수 sanity**: 오프시즌당 성사 건수가 현실적(KBO 연 몇~십몇 건). 리그
   총량/팀당 한도 작동.
2. **니즈 매칭**: 성사 거래의 대부분이 윈나우↔리빌딩 상보 거래인지 —
   윈나우가 유망주/지명권을 내주고 즉전을 받는 방향 샘플. 윈나우↔윈나우는
   드문지(자연 발생 확인).
3. **등가성/결렬**: 성사 거래의 양측 `value_of`(객관값) 합이 근사한지 +
   후려치기 결렬이 실제로 일어난 횟수(시도 대비 성사율).
4. **유망주 도박성**: 리빌딩팀이 받은 유망주의 N년 후 OVR 추적 — 대박과 bust가
   모두 발생하는지 (숨김 재능 → 결과 분산).
5. **★ 순위 순환**: 트레이드 도입 후 30시즌 순환 유지 (4시드+). 트레이드는
   '윈나우 지금 강화 ↔ 리빌딩 미래 강화'의 시간차 재분배라 중립~도움 예상 — 확인.
   깨지면 조정 레버: 팀당 한도 축소 → gm_noise 축소(비합리 거래 감소) → tol 축소.
6. **회귀 가드**: 단일 시즌 경기 지표 불변.

### 승인받을 핵심 결정
1. **성사 3조건** — (a) GM 주관 등가성(tol, 후려치기 결렬) (b) 양쪽 니즈 부합
   (c) 방향 상보성(윈나우↔리빌딩 쌍 우선) — 셋 다 충족해야 성사
2. **GM 주관 평가** — `value_of × exp(N(0, σ~0.20))`, 평가 차이가 거래를 만들고
   숨김 재능이 도박 결과를 가름 (드래프트 노이즈 메커니즘 재사용)
3. **지명권 실물화** — 1~3R 민팅, 라운드별 기대 WAR 테이블로 `value_of` 구현,
   드래프트에서 보유팀이 행사 (슬롯 순서는 원소유팀 위치)
4. **체인**: 에이징 → **트레이드** → FA → 드래프트 → 재정 (트레이드 지명권이
   당해 드래프트에서 바로 행사)
5. 패키지는 소형(베테랑 ↔ 유망주 + 지명권/보조), 팀당·리그 총량 한도
