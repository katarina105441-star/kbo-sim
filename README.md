## 웹 UI 실행 — 단장 모드 (4단계 MVP-1, 설계: [DESIGN_UI.md](DESIGN_UI.md))

아래 두 줄을 순서대로 터미널에 복사해 실행하면 됩니다 (최초 1회 첫 줄, 이후엔 둘째 줄만):

```bash
pip install fastapi uvicorn
python -m uvicorn web.backend.main:app --port 8000
```

브라우저에서 **http://localhost:8000** 접속 → 구단 선택 → 시즌 진행.
(프론트엔드는 빌드본이 저장소에 포함돼 있어 별도 설치가 필요 없습니다.
화면을 고치는 개발자용: `cd web/frontend && npm install && npm run build`)

### 윈도우 처음 설치 가이드 (코딩 몰라도 됨)

1. **파이썬 설치**: https://www.python.org/downloads/ → 노란 "Download Python" 버튼
   → 받은 파일 실행 → **맨 아래 "Add python.exe to PATH" 체크박스를 반드시 체크**
   → Install Now. ("Setup was successful" 화면이 뜨면 성공)
2. **게임 코드 받기**: 이 저장소 GitHub 페이지에서 초록 **[Code] 버튼 → Download ZIP**
   → 받은 ZIP을 우클릭 → "압축 풀기" → 문서 폴더 등에 풀기.
3. **명령창 열기**: 압축 푼 폴더(README.md가 보이는 폴더)를 탐색기로 연 뒤,
   **주소창에 `cmd` 라고 치고 엔터** → 검은 명령창이 그 폴더 위치로 열림.
4. **설치 (최초 1회)**: 명령창에 아래를 붙여넣고 엔터
   ```
   pip install fastapi uvicorn
   ```
   ("Successfully installed ..." 이 뜨면 성공)
5. **게임 실행**: 아래를 붙여넣고 엔터
   ```
   python -m uvicorn web.backend.main:app --port 8000
   ```
   ("Uvicorn running on http://127.0.0.1:8000" 이 뜨면 성공. 이 창은 켜둔 채로 둘 것)
6. **브라우저에서 접속**: 크롬/엣지 주소창에 `localhost:8000` 입력 → 구단 선택 화면.
7. **끝낼 때**: 게임 화면에서 [저장] 누르고, 명령창에서 `Ctrl+C` (다음엔 3→5→6만).

**막힐 때**:
- `'python'은(는) ... 인식할 수 없습니다` → 1번에서 PATH 체크를 안 한 것.
  파이썬 설치 파일을 다시 실행해 체크하고 재설치.
- `'pip'은(는) ... 인식할 수 없습니다` → `python -m pip install fastapi uvicorn` 으로 시도.
- `Address already in use` → 이미 켜져 있음. 열린 명령창을 닫거나 `--port 8001`로
  바꿔 실행하고 브라우저도 `localhost:8001`.
- 브라우저에 "게임이 없습니다" → 정상. 팀을 선택해 새 게임을 시작하면 됨.

MVP-1 "관전하는 단장": 팀 선택 → 시즌 진행(하루/시리즈/월/끝까지) → 대시보드·
순위표·일정/결과(박스스코어)·로스터·선수 상세 열람 + 오프시즌 자동 통과(요약
리포트) + 저장/이어하기. 오프시즌 개입(트레이드/FA/드래프트)과 라인업 편집은 MVP-2.

## 콘솔 실행

```bash
# 단일 경기 (박스스코어)          팀 ID: KIA LG DSN SSG SAM KT LTE HWE NC KWM
python scripts/play_game.py --home KIA --away LG --seed 42
python scripts/play_game.py --home HWE --away KWM --verbose   # 타석별 로그 포함

# 풀시즌 파이프라인 — 정규 144경기 → 순위 → 포스트시즌 → 우승팀 (+PS 개인기록)
python scripts/run_season.py --seed 2026

# 밸런싱 하네스 — N시즌 평균 지표 vs 실제 KBO 목표표
python scripts/calibrate.py --seasons 20 --seed 1

# 분포 검증 — 리그 리더 꼬리 / 포스트시즌 시드별 우승률·언더독 런
python scripts/tail_check.py --seasons 20
python scripts/ps_check.py --seasons 200

# 프랜차이즈 검증 — N시즌 연속 + 오프시즌 에이징 (드리프트/나이곡선/샘플 궤적)
python scripts/franchise_check.py --seasons 12

# 테스트
python -m unittest discover -s tests
```

## 구조

```
data/       실명 선수 250명(팀당 25) + 구단 메타. 능력치는 실제 성적 역산 (est=추정 표기)
kbo/models  Player / Team / 기록 (순수 데이터)
kbo/engine  probability(log5 수식+TUNE) → plate_appearance(타석 트리)
            → baserunning / defense / pitching_manager → game(경기 러너)
kbo/league  일정 + 시즌 러너 (불펜 연투 제한 포함)
kbo/io      로더 + 콘솔 리포트 (print는 여기서만)
scripts/    play_game / run_season / calibrate
```

- 엔진은 print 없음, `random.Random(seed)` 주입 → 같은 시드 = 같은 경기
- 모든 밸런싱 상수는 `kbo/engine/probability.py`의 `TUNE` 한 곳
- 파크팩터는 전 구장 1.0(중립) 뼈대만 — 값만 채우면 작동

## 포스트시즌 (계단식 토너먼트)

WC(4위 1승 어드밴티지) → 준PO(5전3선승) → PO(5전3선승) → KS(7전4선승, 홈 2-3-2).
단기전 지정 선발(에이스 중3일 재등판), 총력전 불펜, 시리즈 간 휴식,

검증(350시즌): 1위 우승률 57.5~58.7% (실제 ~50-60%) · 5위 언더독 KS 진출 5.5~7.3% ·
중3일 선발 승률 열세·연투 불펜 22% 투입 확인.

## 에이징 커브 (3단계 선행, 설계: [DESIGN_AGING.md](DESIGN_AGING.md))

+ 숨김 재능 g/d + 연간 노이즈. 은퇴 3단 규칙 → 1:1 스텁 신인(정식 드래프트 전 임시).

## 알려진 이슈 (KNOWN ISSUES)

### 해결됨
- **도루왕 고갈** → 아키타입 신인 시스템으로 해결. 60시즌 검증에서 평균 42
  (실제 35~64), 붕괴(20 미만) 0회 (이전 최소 7).
- **엘리트 재생산** — 홈런왕·탈삼진왕 등 상단 꼬리가 30시즌 장기 시뮬 후에도
  유지됨을 확인 (스타 은퇴에도 리그가 밋밋해지지 않음).
- **타율왕 상한 비대화(컨택 특화형)** → 드래프트 단계에서 아키타입 컨택/선구
  시그니처 하향(`sig_mean_override` 79)으로 해결. **4시드×15시즌** 재검증에서
  .390+ 3/60(5%)·후반 1/32로 상시화 소멸, 홈런왕(평균 40)·도루왕(평균 40)
  특화 재생산은 유지 (균형점 확보).

### 미결
1. **드리프트 경계**: 장기 모드에서 타율 +0.006~0.012 / ERA +0.20~0.46.
   단일 시즌 밸런싱은 안 깨짐(테스트 가드), **장기 모드에서만** 나타남.
   (연봉 인플레는 `war_growth_damp`로 캡에 정렬 완료 — DESIGN_CONTRACTS.md §7.)
2. **측정 방법론**: TUNE 변경 시 RNG 스트림이 이동해 같은 시드도 다른 난수
   우주가 됨 → 1%p 수준 효과 측정엔 **4시드+ 필요**. 장기 튜닝은 반드시
   4시드 이상 하네스로 판단할 것.

### 백로그 (나중에 '구단 개성' 살릴 때)
- **감독/구단 스카우팅 성향 편향**: 구단마다 파워/컨택/주루 등 능력치 선호가
  달라 드래프트·FA에서 편향된 선택을 하게. (실제: 김경문 감독이 발 빠른 야수를
  선호해 스카우트팀에 의사 전달한 사례.) 현재 드래프트는 라운드별 BPA/Need만
  차등 — 구단별 선호 편향은 미구현. 트레이드/구단운영 확장 시 추가.

## 웹 사용자 개입 현황 (MVP-3)

- 라인업·수비 슬롯·선발 로테이션·마무리·셋업 직접 편집
- 실시간 경기 한 타석 진행 + 수동 투수 교체
- 대타·대주자·대수비 직접 지시와 진행 중 경기 저장·복원
- 오프시즌 FA 직접 입찰: 선수별 AAV 제안·패스·AI 판단, 예산·보상금·영입 한도 반영
- 오프시즌 신인 드래프트 직접 지명: AI 지명 자동 진행, 사용자 지명권에서 정지,
  스카우팅 오차가 적용된 보드와 AI 추천 제공
- 후속 범위: 트레이드 제안·수락, 2군/육성

  초안 σ0.20에서 ~25% → 0.12로 조임)·유망주 도박 창발·순위 순환 유지(우승 0회

남은 확장: 트레이드 직접 협상·2군/육성·보상선수·구단 개성.
